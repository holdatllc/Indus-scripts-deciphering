# A Structural Constraint Model of the Indus Script

## Overview
This project presents a **structural constraint model** of the Indus script derived from the Miller Harmonic Mathematics (MHM) audit pipeline.

Rather than attempting linguistic decipherment, this work focuses strictly on **formal structure**, identifying reproducible positional, relational, and segmentation constraints across a corpus of **n = 1,670 inscriptions**.

## Core Contribution
We demonstrate that the Indus script can be modeled as a **four-layer constraint system**:

1. **Iconographic Motif Class (Contextual Layer)**
   - External visual domain (e.g., Unicorn, Zebu Bull)
   - Provides contextual grouping for inscriptions

2. **Core Identity Layer**
   - High-stability symbols (e.g., M178, M329)
   - Represent statistically stable identity units

3. **Functional Class Layer (A/B)**
   - Behavior-based classification derived from positional constraints
   - Examples:
     - M024 → observed exclusively in unicorn-associated contexts
     - M056 → occurs across multiple motif classes
     - M062 → observed exclusively in zebu bull contexts

4. **Syntactic Boundary Layer**
   - M125 identified as an **empirically validated boundary operator**
   - Enables segmentation into independent clauses

## Key Results

- **99.76% structural consistency**
  - One CORE identity per clause (Phase 6 validation)

- **100% independent segment validity**
  - Verified using M125 segmentation (Phase 5)

- **Deterministic positional roles**
  - Symbols cluster into functional zones (CORE, REL, CL)

## Important Scope Limitation

> All role assignments are derived from positional, transition, and stability constraints.  
> **No phonetic or semantic interpretation is implied.**

This model does **not claim decipherment**. It defines the structural rules governing inscription construction.

## Interpretation

This work reframes the Indus script as a **constraint-driven encoding system**, where:

- Motif → contextual domain  
- CORE → identity unit  
- Functional class → behavioral modifier  
- Boundary → clause delimiter
